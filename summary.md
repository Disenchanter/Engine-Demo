# 作品説明資料（Summary）

## 1. 開発にかかった期間
- 開発期間（概算）：約 5 週間
- 実作業日数（概算）：10 日

## 2. 所要時間数の概算
- 総作業時間（概算）：約 30 時間
  - レンダリング基盤整備：10 時間
  - PBR/Phong マテリアル実装と調整：8 時間
  - ECS 連携（Transform 同期・生成処理）：5 時間
  - UI（ImGui）と動作確認・修正：5 時間

## 3. 開発メンバー数と構成
- メンバー数：1 名
- 構成：
  - クライアントアプリ実装（C++ / OpenGL）
  - シェーダー実装（GLSL）
  - ビルド環境整備（CMake + Ninja）

## 4. 自身の担当範囲
- シーン管理・描画処理の実装
- PBR/Phong マテリアル分岐とパラメータ連携
- ECS のメッシュ/Transform 同期処理
- ImGui によるオブジェクト追加 UI の実装
- デバッグ、表示不具合修正、README 整備

## 5. 動作環境
- OS：Windows 10 / 11
- 言語：C++17
- ビルド：CMake 3.12 以上、Ninja
- 描画：OpenGL（GLFW + GLAD）
- UI：ImGui

## 6. 操作方法
1. アプリを起動
2. `Scene Controls` パネルで `Shape Type` を選択（Box / Sphere）
3. `Position X/Y/Z` と `Size` を調整
4. `Create Object` でシーンに追加
5. `Clear Color` で背景色を変更

## 7. 制作意図の解説
本作品は、**リアルタイム 3D 描画の学習**を目的として制作した。
特に以下を重視した：
- マテリアル差分（PBR / Phong）を視覚的に比較しやすい構成
- ECS とシーングラフを併用した、最小構成の拡張しやすい設計
- ImGui で動作確認しやすい実験環境の提供

また、単一シーン上で Box / Sphere の材質差を即時確認できるようにし、
シェーダーやライティング調整の試行回数を増やせる開発体験を狙った。
